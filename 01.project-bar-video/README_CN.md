让我们先回答几个关键问题：

### 问题：数据和材料从哪里来？

**答案：**

- 导出 `TTM(bounded)` 工作表为 `.csv` 格式获取原始数据。将其重命名为 `airlines_final.csv` 并放置在 [../99.utility/airline-bar-video](../99.utility/airline-bar-video) 文件夹中
  ![数据导出过程](https://drive.google.com/PLACEHOLDER_DATA_EXPORT.png)

  仅供参考：TTM（12个月滚动汇总）是将年度数据除以4生成季度数据，然后汇总最近4个季度。如果需要验证数据，请查看 `Quarterly Revenue&EBITDA` 和 `Raw: Annual Revenue/EBITDA`。
- 使用 `Timeline` 工作表获取新闻评论和图片

### 问题：我只需要拼接最新的两个季度视频吗？如何保持材料组合的一致性？

**答案：** 浏览器几乎无法生成完全相同的材料（受颜色、亮度、锯齿影响），所以我们不能只更新最新的两个季度材料，而是必须从头开始录制每个组件并一起编辑。当然，Ned也可能对之前的材料提出新的修改要求————重新做一遍是不可避免的。但是，对于之前的新闻图文，我们的确可以偷偷懒，用色度抠图抠出来（详见下面一个问题），而不用自己做。

为了保持一致性，请使用 **Eloquia Display Regular** 作为字体（我猜的）。但是这个字体要钱，如果我们不想付钱，请下载**Eloquia Display Extra Bold**和**Eloquia Text Extra Light**。可惜，加粗的extra light并不能完全模仿regular，尽力了。

### 问题：如果我缺少某些材料怎么办？

**答案：** 可以是前一个视频的片段、冻结帧或截图。我们可以使用色度抠图（即抠绿幕）的方法去除灰色网格线。一些新闻材料就是通过这种方法获得的。

![色度键处理过程](https://drive.google.com/PLACEHOLDER_CHROMA_KEY.png)

**重要注意事项：**

- 色度键需要正确的色度、边缘羽化和模糊设置
- 色度过高会额外去除新闻截图中的灰色，让图片破碎
- 边缘羽化或模糊过高会使整个图像模糊（为了去除网格线，一些模糊是不可避免的）
- 逐步增加色度、羽化和模糊，直到网格线被去除
- **关键：** 判断网格线是否被去除的标准不是视频编辑软件的预览（因为预览是低分辨率的）。我们必须留有余量，并检查最终导出的视频中是否能看到网格线。

### 问题：视频编辑时有没有规范？

**答案：** 图文可以这样设置：总时长4s，淡入淡出各1秒，让淡入的时间点大约在数据发生相应变化（大幅移动、logo变化）的时点。请注意，我们不需要严格按现实时间排布新闻，重要的是把Ned希望展示的新闻全部塞进来的同时，尽量保证新闻能解释当时的数据变化。如果某一时段新闻较多，可以让他们在视频中前后移动几个月，也可以适当缩短展示时长。

## 代码修改指南

1. **加粗线条宽度防止闪烁**：任何元素的像素都不能太少，否则浏览器的抗锯齿功能会模糊边缘像素，导致明显的粗细变化和闪烁
2. **航空公司的显示名称和唯一名称**

请注意我们现在是如何处理航司合并的。在`TTM(bounded)` 工作表中，我们已经人为删除了发生合并的小航司的数据，只把数据记录在大航司的列中。比如，`AF`和`KL`在2004年5月合并为`AF/KL`，那么`AF`和`KL`列始终都没有数据，而`AF/KL`列在合并的时点前，显示的数据实际上是之前营收更多的航司`AF`。因此，数据是连续的，变化是明显的，我们只需要在适当的时间修改显示的IATA code和logo即可。在这里，`AF/KL`是数据的唯一名称（column name/index），而之前的`AF`和之后的`AF/KL`是我们设置的显示名称。

不过，因为我们只有季度数据，代码把月份映射为3、6、9、12月，所以如果设置`大于2004年5月时切换为AF/KL`，那么在第二季度动画开始时，即4月，就会因为第二季度的月份是6月，满足`大于2004年5月`，而看到显示名称的变化。为了避免这种变歧义，我这里人为后移了一个季度，避免在合并前就显示。

FYI，这种设置一是为了简化代码，二是因为，合并而成的航司并不是一个新入局的角色，所以不应该表现为小航司退出市场（减少到0）、大航司进入市场（从0增长），而是应该在变化的时点直接“摘桃子”原地变身。区分显示名称和唯一名称是我认为最简单的方法。唯一的缺点是，当前没有另外写一个 .js 文件来记录这种变化，而是硬编码在代码中。
