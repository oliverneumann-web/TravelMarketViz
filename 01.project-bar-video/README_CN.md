### 问题：我只需要拼接最新的两个季度视频吗？如何保持材料组合的一致性？

**答案：** 浏览器几乎无法生成完全相同的材料（受颜色、亮度、锯齿影响），所以我们不能只更新最新的两个季度材料加到视频最后，而是必须从头开始录制每个组件并一起编辑。当然，Ned也可能对之前的材料提出新的修改要求————重新做一遍是不可避免的。但是，对于之前的新闻图文，我们的确可以偷偷懒——用色度抠图抠出来（详见下面一个问题），而不用自己做。

为了保持一致性，请使用 **Eloquia Display Regular** 作为字体（我猜的）。但是这个字体要钱。如果我们不想付钱，请下载**Eloquia Display Extra Bold**和**Eloquia Text Extra Light**。可惜，加粗的extra light并不能完全模仿regular，尽力了。

### 问题：如果我缺少某些材料怎么办？

**答案：** 可以是前一个视频的片段、冻结帧或截图。我们可以使用色度抠图（即抠绿幕）的方法去除灰色网格线。一些新闻材料就是通过这种方法获得的。

![色度键处理过程](https://drive.google.com/PLACEHOLDER_CHROMA_KEY.png)

**重要注意事项：**

- 色度键需要正确的色度、边缘羽化和模糊设置
- 色度过高会额外去除新闻截图中的灰色，让图片破碎
- 边缘羽化或模糊过高会使整个图像模糊（为了去除网格线，一些模糊是不可避免的）
- 逐步增加色度、羽化和模糊，直到网格线被去除
- **关键：** 判断网格线是否被去除的标准不是视频编辑软件的预览（因为预览是低分辨率的）。我们必须留有余量，并检查**最终导出的视频中**是否能看到网格线。

### 问题：视频编辑有没有规范？

**答案：** 图文可以这样设置：总时长4s，淡入淡出各1秒，让淡入完成的时间点（亮度100%）大约在数据发生相应变化（大幅移动、logo变化）的时点。请注意，我们不需要严格按真实日期排布新闻，重要的是把Ned希望展示的新闻全部塞进来的同时，尽量保证**新闻能解释当时的数据变化**。如果某一时段新闻较多，可以让他们在视频中前后移动几个月，也可以适当缩短展示时长。

## 代码修改指南

1. **加粗线条宽度防止闪烁**：任何元素的像素都不能太少，否则浏览器的抗锯齿功能会模糊边缘像素，导致明显的粗细变化和闪烁
2. **航空公司的显示名称和唯一名称**

请注意我们现在是如何处理**航司合并**的。在源数据`TTM(bounded)` 工作表中，我们已经人为删除了发生合并的小航司的数据，只把数据记录在大航司的列中。比如，`AF`和`KL`在2004年5月合并为`AF/KL`，那么`AF`和`KL`列始终都没有数据，而`AF/KL`列在合并的时点前，显示的数据实际上是之前营收更多的航司`AF`。因此，数据是连续的，变化是明显的，我们只需要在适当的时间修改显示的IATA code和logo即可。在这里，`AF/KL`是数据的唯一名称（column name/index），而之前的`AF`和之后的`AF/KL`是我们设置的显示名称。

不过，因为我们只有季度数据，代码把月份映射为3、6、9、12月，所以如果设置`大于2004年5月时切换为AF/KL`，那么在第二季度动画开始时，即4月，就会因为第二季度的月份是6月，满足`大于2004年5月`，而看到显示名称的变化。为了避免这种变歧义，我这里人为后移了一个季度，避免在合并前就显示。

FYI，这种设置一是为了简化代码，二是因为，合并而成的航司并不是一个新入局的角色，所以不应该表现为小航司退出市场（减少到0）、大航司进入市场（从0增长），而是应该在变化的时点直接“摘桃子”原地变身。区分显示名称和唯一名称是我认为最简单的方法。唯一的缺点是，为了减少对代码的改动，我没有选择另外写一个 `.js` 文件来记录这种变化，而是硬编码在函数`update_display_names_for_quarter`中。

3. **bar chart逻辑速览**

整个动画可以理解为两个季度绘制的两帧间的插值帧。我们没有设置复杂的逻辑，而是简单粗暴地重绘每一帧，在每一帧内都进行了排名。因此，整个html非常大，本质上就是用`.py`生成了硬编码每一帧的`.html`（包括logo）。在大多数情况下，不要尝试推翻这个逻辑，我也不知道会发生什么。

我修复了一个bug：这个bug曾导致，帧间插值基于两个帧各自的“第n名”，并在季度切换时刷新“第n名”的名称和logo，而不是基于航空公司自己的数进行插值。比如，`AF`从1增长到4，而`BA`从2增长到3，动画会让第一名的bar从2慢慢增长到4，并在季度中插值时始终显示`AF`，最后等到完全增长到4时，再切换为`BA`。这让季度间的动画实际上并不对应显示的航司，并导致非常明显的“跳变”（排名瞬间大幅变化，且没有两个bar相遇的中间状态）。现在，bar会正确地基于航空公司自己的数据进行增长和移动。我认为代码不再会导致这种“跳变”；如有，请先查看是否是源数据的错误（异常数据会呈现异常排名，在视觉上近似“跳变”）。

4. **测试阶段和生产阶段**

在测试阶段，我们可以搜索`const quarterDuration`并把它修改为200（单位是ms），以便快速预览。在生产阶段，请恢复为默认值3000。这能保证较丝滑的动画，并给我们留出更多前置时间打开录屏软件。绘制完毕后，将自动用浏览器打开`.html`文件。我们也可以在文件夹中双击该文件打开。

请注意，如果我们使用的是Nvidia或自动窗口采集，不要在代码编辑器页面就开始录制并等待自动打开网页——录屏会锁定编辑器窗口。必须等待网页打开以后，再按 Win+G 开启录屏。代码将从1997开始绘制，我们只需要在2000前完成录屏软件的启动即可。

5. **logo的显示**

logo显示有两个逻辑：一是在不同时间展示对应的不同的logo，二是在视觉上让每个logo（logo里的文字）差不多大。因此，logo的工作流比较细致：
1. 在[超链接https://logos-world.net/]搜索并下载logo图片，在`logo_mapping`填好适用年份和路径（代码会在`start year`的第一个季度就使用新的logo，不够准确，但是比较麻烦我没改）
2. 在填写的时候注意，老图片是`.png`，而最新的图片往往是`.jpeg`。可能是新图片复用了头图的素材，有不同的逻辑。写路径的时候检查一下，不要写错了。
3. 用[超链接https://www.photopea.com/]裁剪图片并去除背景（因为动画背景是白色的，所以裁剪好了一般不需要去除背景，如果去除了背景一定要另存为`.png`）。裁剪掉白边是为了让logo尽量贴合真实的logo图案。[图片超链接]
4. 针对logo图标、文字的真实布局，调整`logo_sizes`，让logo看起来差不多大。比如Emirates，就是字小所以调大一点。
